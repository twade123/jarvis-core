## 7 Systemic Issues Found (Priority Ordered)

### 1. Story=0 Alerts Should Be Suppressed [EASY FIX]
- **Impact**: Eliminates ~16% of garbage alerts
- **What**: 8 alerts fired with story_score=0. Zero-story = zero narrative = no alert.
- **Fix**: `if story_score < 15: skip_alert()`

### 2. Chop Zone Filter [EASY FIX]
- **Impact**: Kills GBP_USD-type noise, ~8% of alerts
- **What**: GBP_USD with 0.4 pip fan separation and tangled EMAs gets alerts
- **Fix**: `if fan_sep_pips < 2.0 AND bb_trend == 'CONTRACTING': suppress`

### 3. Alert Cooldown / Dedup [MEDIUM]
- **Impact**: Cuts alert volume ~60% (AUD_USD: 11 → ~3)
- **What**: Same pair+direction alerts every 30 min scan cycle
- **Fix**: Don't re-alert same pair+direction within 2 hours unless Δstory ≥ 15

### 4. Expansion Maturity Check [MEDIUM]
- **Impact**: Prevents top-calling (EUR_USD 05:01, EUR_JPY trade)
- **What**: "Early expansion" called on mature/exhausting fans
- **Fix**: Count bars since fan ordering began + check price vs BB position. If ordered 30+ bars AND price at BB extreme → "late expansion" label, lower priority.

### 5. Coherence Gate [MEDIUM]
- **Impact**: Eliminates contradictory calls (USD_CHF, USD_JPY, NZD_USD)
- **What**: Both LONG expansion and SHORT reversal on same pair within hours
- **Fix**: If previous alert same pair was opposite direction AND fan state unchanged → suppress

### 6. Fan Delta Sign Check [EASY FIX]
- **Impact**: Fixes misleading "developing" labels (EUR_AUD)
- **What**: Scout calls compressing fan "bullish fan developing" because EMAs are still ordered
- **Fix**: Check fan_delta_5bar: if negative for bullish (positive for bearish) → "compressing" not "developing"

### 7. "No Clear Opportunity" Should Not Create Alert Rows [EASY FIX]
- **Impact**: Reduces DB noise, saves validator cycles
- **What**: "No clear opportunity — waiting for setup" logged as scout_alerts rows
- **Fix**: Log these as scout_scans (telemetry) not scout_alerts (actionable)
